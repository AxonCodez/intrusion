1.  Introduction  1.1  Background  and  Motivation  
The  digital  transformation  of  modern  enterprises,  coupled  with  the  exponential  growth  of  
Internet
 
of
 
Things
 
(IoT)
 
devices,
 
cloud-based
 
services,
 
and
 
distributed
 
computing
 
architectures,
 
has
 
fundamentally
 
altered
 
the
 
cybersecurity
 
landscape.
 
Contemporary
 
networks
 
generate
 
massive
 
volumes
 
of
 
traffic
 
every
 
second—often
 
measured
 
in
 
terabytes
 
per
 
day
 
for
 
large-scale
 
organizations—making
 
manual
 
monitoring
 
by
 
security
 
analysts
 
not
 
only
 
impractical
 
but
 
fundamentally
 
ineffective
 
against
 
the
 
speed
 
and
 
sophistication
 
of
 
modern
 
cyber
 
threats.
 
According
 
to
 
recent
 
industry
 
reports,
 
the
 
global
 
cost
 
of
 
cybercrime
 
is
 
projected
 
to
 
exceed
 
$10.5
 
trillion
 
annually
 
by
 
2025,
 
with
 
network
 
intrusion
 
attacks
 
constituting
 
one
 
of
 
the
 
most
 
prevalent
 
and
 
financially
 
devastating
 
attack
 
vectors.
   
Intrusion  Detection  Systems  (IDS)  have  emerged  as  a  foundational  component  of  layered  
network
 
security
 
architectures,
 
serving
 
as
 
the
 
digital
 
equivalent
 
of
 
surveillance
 
cameras
 
for
 
network
 
traffic.
 
An
 
IDS
 
monitors
 
network
 
packets,
 
system
 
logs,
 
and
 
flow
 
data
 
in
 
real-time
 
or
 
near-real-time,
 
applying
 
analytical
 
techniques
 
to
 
identify
 
patterns
 
indicative
 
of
 
malicious
 
activity.
 
Upon
 
detection,
 
these
 
systems
 
generate
 
alerts
 
for
 
security
 
operations
 
center
 
(SOC)
 
personnel
 
or
 
trigger
 
automated
 
response
 
mechanisms
 
through
 
integration
 
with
 
Intrusion
 
Prevention
 
Systems
 
(IPS).
 
The
 
efficacy
 
of
 
an
 
IDS
 
directly
 
influences
 
an
 
organization's
 
ability
 
to
 
detect,
 
respond
 
to,
 
and
 
recover
 
from
 
security
 
incidents
 
before
 
they
 
escalate
 
into
 
catastrophic
 
breaches.
   
Traditional  IDS  architectures  have  predominantly  relied  on  signature-based  detection  
methodologies,
 
wherein
 
network
 
traffic
 
is
 
compared
 
against
 
a
 
database
 
of
 
known
 
attack
 
patterns  or  "signatures."  While  effective  against  previously  cataloged  threats,  signature-based  
systems
 
exhibit
 
fundamental
 
limitations:
 
they
 
cannot
 
detect
 
zero-day
 
attacks
 
for
 
which
 
no
 
signature
 
exists,
 
they
 
require
 
continuous
 
database
 
updates
 
that
 
introduce
 
latency
 
between
 
threat
 
emergence
 
and
 
detection
 
capability,
 
and
 
they
 
are
 
susceptible
 
to
 
evasion
 
techniques
 
such
 
as
 
polymorphic
 
malware
 
and
 
traffic
 
obfuscation.
 
These
 
limitations
 
have
 
catalyzed
 
a
 
significant
 
research
 
shift
 
toward
 
anomaly-based
 
and
 
machine
 
learning-driven
 
detection
 
approaches.
   
Machine  learning-based  IDS  analyze  traffic  behavior  patterns  and  learn  to  distinguish  
between
 
benign
 
and
 
malicious
 
activities
 
based
 
on
 
statistical
 
models
 
derived
 
from
 
training
 
data,
 
rather
 
than
 
relying
 
on
 
static
 
signature
 
matching.
 
This
 
paradigm
 
offers
 
several
 
compelling
 
advantages:
 
the
 
ability
 
to
 
detect
 
previously
 
unseen
 
attacks
 
through
 
anomaly
 
identification,
 
adaptability
 
to
 
evolving
 
threat
 
landscapes
 
through
 
model
 
retraining,
 
and
 
the
 
capacity
 
to
 
process
 
high-dimensional
 
feature
 
spaces
 
that
 
capture
 
nuanced
 
traffic
 
characteristics.
 
The
 
effectiveness
 
of
 
such
 
systems,
 
however,
 
is
 
critically
 
dependent
 
on
 
the
 
quality,
 
diversity,
 
and
 
realism
 
of
 
the
 
benchmark
 
datasets
 
used
 
for
 
training
 
and
 
evaluation.
   
1.2  The  CICIDS2017  Dataset  
Among  publicly  available  network  intrusion  detection  datasets,  the  Canadian  Institute  for  
Cybersecurity
 
Intrusion
 
Detection
 
System
 
2017
 
(CICIDS2017)
 
dataset
 
has
 
achieved
 
widespread
 
recognition
 
as
 
a
 
gold
 
standard
 
for
 
IDS
 
research
 
and
 
evaluation.
 
Developed
 
by
 
Sharafaldin,
 
Lashkari,
 
and
 
Ghorbani
 
at
 
the
 
University
 
of
 
New
 
Brunswick,
 
the
 
CICIDS2017
 
dataset
 
was
 
generated
 
over
 
a
 
five-day
 
period
 
(Monday
 
through
 
Friday)
 
in
 
a
 
controlled
 
but
 
realistic
 
network
 
environment
 
that
 
closely
 
emulates
 
modern
 
enterprise
 
network
 
topologies.
   
The  dataset  encompasses  both  benign  background  traffic  and  multiple  contemporary  attack  
categories,
 
representing
 
one
 
of
 
the
 
most
 
comprehensive
 
and
 
realistic
 
publicly
 
available
 
IDS
 
benchmarks.
 
Key
 
characteristics
 
that
 
distinguish
 
CICIDS2017
 
from
 
predecessors
 
include:
   
●  Realistic  traffic  profiles:  Benign  traffic  was  generated  using  automated  profiles  that  
simulate
 
real
 
user
 
behavior,
 
including
 
HTTP
 
browsing,
 
HTTPS
 
browsing,
 
email,
 
FTP,
 
SSH,
 
and
 
DNS
 
queries.
 ●  Comprehensive  attack  diversity:  The  dataset  includes  DoS,  DDoS,  Brute  Force  
(FTP-Patator,
 
SSH-Patator),
 
Web
 
Attacks
 
(Brute
 
Force,
 
XSS,
 
SQL
 
Injection),
 
Infiltration,
 
Botnet,
 
and
 
PortScan
 
activities.
 ●  Flow-based  features:  Network  traffic  is  represented  using  80+  statistical  flow  features  
extracted
 
using
 
CICFlowMeter,
 
including
 
packet
 
counts,
 
byte
 
volumes,
 
inter-arrival
 
times,
 
flag
 
statistics,
 
and
 
window
 
sizes—eliminating
 
the
 
need
 
for
 
raw
 
packet
 
inspection.
 ●  Labeled  data:  Each  flow  record  is  labeled  with  its  corresponding  attack  category,  enabling  
supervised
 
learning
 
approaches.
 ●  Temporal  separation:  Training  and  test  data  are  temporally  separated  (different  days),  
reducing
 
the
 
risk
 
of
 
temporal
 
overfitting
 
and
 
providing
 
a
 
more
 
realistic
 
evaluation
 
scenario.

2.1  Machine  Learning-Based  Detection  of  Web  Attacks  
2.1.1  Attack  Characteristics  
The  CICIDS2017  dataset  includes  three  application-layer  web  attack  subtypes:  Web  Attack  –  
Brute
 
Force,
 
Web
 
Attack
 
–
 
XSS
 
(Cross-Site
 
Scripting),
 
and
 
Web
 
Attack
 
–
 
SQL
 
Injection.
 
These
 
attacks
 
differ
 
fundamentally
 
from
 
volumetric
 
threats
 
such
 
as
 
DDoS
 
in
 
their
 
operational
 
characteristics:
   
●  Web  Attack  –  Brute  Force:  Involves  automated  repeated  authentication  attempts  against  
web
 
application
 
login
 
endpoints.
 
While
 
generating
 
multiple
 
HTTP
 
requests,
 
the
 
traffic
 
volume
 
remains
 
relatively
 
low
 
compared
 
to
 
flooding
 
attacks.
 
The
 
distinguishing
 
characteristics
 
include
 
repetitive
 
POST
 
requests
 
to
 
authentication
 
endpoints,
 
consistent
 
inter-arrival
 
times
 
between
 
requests,
 
and
 
predictable
 
packet
 
payloads.
 ●  Web  Attack  –  XSS  (Cross-Site  Scripting):  Involves  injection  of  malicious  client-side  
scripts
 
into
 
web
 
pages
 
viewed
 
by
 
other
 
users.
 
From
 
a
 
network
 
flow
 
perspective,
 
XSS
 
attacks
 
often
 
manifest
 
as
 
single
 
or
 
few
 
HTTP
 
requests
 
containing
 
malicious
 
payloads
 
in
 
parameters.
 
The
 
flow-level
 
statistical
 
features
 
of
 
XSS
 
traffic
 
closely
 
resemble
 
benign
 
web
 
browsing
 
traffic,
 
making
 
detection
 
particularly
 
challenging.
 ●  Web  Attack  –  SQL  Injection:  Involves  injection  of  malicious  SQL  queries  through  web  
application
 
input
 
fields.
 
Similar
 
to
 
XSS,
 
SQL
 
Injection
 
attacks
 
typically
 
generate
 
minimal
 
traffic
 
volume,
 
with
 
malicious
 
payloads
 
embedded
 
within
 
normal-looking
 
HTTP
 
requests.
 
The
 
flow-based
 
statistical
 
features
 
of
 
SQL
 
Injection
 
traffic
 
exhibit
 
significant
 
overlap
 
with
 
legitimate
 
database-driven
 
web
 
application
 
traffic.
   
The  fundamental  detection  challenge  for  web  attacks  lies  in  their  subtlety:  unlike  volumetric  
attacks
 
that
 
generate
 
obvious
 
statistical
 
anomalies,
 
web
 
attacks
 
introduce
 
nuanced
 
behavioral
 
changes
 
that
 
may
 
be
 
indistinguishable
 
from
 
benign
 
traffic
 
when
 
examined
 
using
 
only
 
flow-based
 
statistical
 
features.
   
2.1.2  Decision  Tree-Based  Approaches  
Decision  Tree-based  classifiers  have  been  extensively  explored  for  web  attack  detection  due  
to

interpretability,
 
low
 
computational
 
overhead,
 
and
 
ability

Ahmim  et  al.  (2018)  proposed  a  novel  hierarchical  intrusion  detection  framework  combining  
Decision
 
Tree
 
and
 
rule-based
 
models,

that
 
tree-based
 
classifiers
 
can
 
effectively
 
distinguish

traffic
 
patterns
 
in
 
complex

environments.
 
Their
 
hierarchical
 
approach
 
decomposed

classification

into
 
stages,
 
first
 
separating
 
benign
 
from
 
malicious
 
traffic,
 
then
 
further
 
classifying
 
attack
 
types.
 
The
 
framework
 
achieved
 
an
 
overall
 
accuracy
 
of
 
96.65%
 
with
 
a
 
detection
 
rate
 
of
 
94.47%,
 
though
 
the
 
hierarchical
 
design
 
increased
 
model
 
complexity
 
and
 
introduced
 
cascading
 
classification
 
errors
 
at
 
each
 
stage.
   
Jumayev  and  Jumayev  (2023)  focused  specifically  on  improving  brute-force  attack  detection  
using
 
Decision
 
Trees,
 
conducting
 
an
 
in-depth
 
analysis
 
of
 
how
 
feature
 
selection
 
and
 
dataset
 
balancing  strategies  influence  classification  performance.  Their  study  highlighted  that  
brute-force
 
attacks,
 
with
 
their
 
repetitive
 
authentication
 
patterns,
 
are
 
more
 
amenable
 
to
 
Decision
 
Tree
 
classification
 
compared
 
to
 
XSS
 
and
 
SQL
 
Injection,
 
which
 
exhibit
 
subtler
 
flow-level
 
deviations
 
from
 
benign
 
traffic.
   
2.1.3  Ensemble  Methods  
Beyond  standalone  Decision  Trees,  ensemble-based  approaches  have  demonstrated  
significantly

performance
 
in
 
web
 
attack

scenarios.
 
Ensemble

combine
 
multiple
 
base

to
 
produce
 
a
 
composite
 
model
 
with
 
reduced
 
variance,
 
lower
 
overfitting

and
 
improved

Vadhil  et  al.  (2023)  reported  high  detection  accuracy  when  applying  Random  Forest  
classifiers
 
to
 
web-based
 
intrusion
 
detection,
 
achieving
 
accuracy
 
exceeding
 
99%
 
in
 
optimized
 
configurations.
 
Random
 
Forest's

to

predictions
 
across

of

each

feature

that

might

Khan  et  al.  (2024)  conducted  balanced  multi-class  network  intrusion  detection  experiments  
using
 
multiple
 
classifiers
 
including
 
Decision
 
Tree,
 
Random
 
Forest,
 
XGBoost,
 
Naive
 
Bayes,
 
Logistic
 
Regression,
 
and
 
AdaBoost.
 
Their
 
results
 
demonstrated
 
that
 
Decision
 
Tree
 
achieved
 
99.96%
 
accuracy
 
in
 
binary
 
classification
 
(Benign
 
vs.
 
Attack)
 
but
 
dropped
 
to
 
96.37%
 
in
 
multi-class
 
settings
 
where
 
web
 
attacks
 
must
 
be
 
distinguished
 
from
 
other
 
attack
 
categories—a

that
 
underscores

impact
 
of
 
classification
 
granularity
 
on
 
reported

2.1.4  Support  Vector  Machines  
Support  Vector  Machines  (SVM)  have  been  applied  to  intrusion  detection  using  the  CICIDS  
dataset

Abuali

(2023)

reporting
 
100%

with
 
perfect

and

However,

of

representativeness

Xu  and  Liu  (2025)  evaluated  multiple  machine  learning  models  on  CICIDS2017  and

margin-based

SVM
 
perform
 
competitively
 
in

but

multi-class
 
scenarios

subtle
 
web
 
attacks.
 
The
 
linear
 
decision
 
boundaries
 
imposed
 
by
 
standard
 
SVM
 
formulations
 
may
 
be
 
insufficient
 
to
 
separate
 
overlapping
 
attack
 
classes
 
in
 
high-dimensional
 
feature
 
spaces.
   
2.1.5  Distance-Based  Methods  
Distance-based  methods  such  as  k-Nearest  Neighbors  (k-NN)  have  been  explored  in  intrusion  
detection
 
research,
 
though
 
their
 
performance
 
is
 
notably
 
sensitive
 
to
 
preprocessing
 
choices.
 
Bao
 
and
 
Gao
 
(2025)
 
proposed
 
an
 
improved
 
k-NN

for
 
network

However,

to
 
feature

in

Additionally,

during

(requiring

poses

high-throughput

2.1.6  Deep  Learning  and  Hybrid  Approaches  
More  recently,  deep  learning  approaches  have  gained  traction  for  web  attack  detection.  Kim  
et
 
al.
 
(2019)

Convolutional
 
Neural

(CNN)

While

Bandarupalli  (2025)  explored  Artificial  Neural  Networks  (ANN/MLP)  architectures  for

Abdelhaq  et  al.  (2026)  proposed  a  hybrid  XGBoost–SVM  ensemble  framework  originally

of

(IoMT)

The

XGBoost's

to

While

2.2  Detection  Methods  for  Infiltration  Attacks  2.2.1  Attack  Characteristics

2.3  Denial  of  Service  (DoS)  Attacks  and  Mitigation  Techniques  2.3.1  Attack  Characteristics  
Denial  of  Service  (DoS)  attacks  aim  to  render  a  server,  service,  or  network  resource

Unlike

The

DoS  Hulk:  An  HTTP  flood  attack  that  overwhelms  web  servers  with  a  massive  number  of  
HTTP

and

to

maintaining
 
high

that
 
saturate
 
server
 
processing

The

is

sudden,

HTTP
 
traffic
 
from
 
a
 
single
 
source
 
IP
 
with
 
randomized
 
request
 
characteristics.
   
DoS  GoldenEye:  An  HTTP-based  attack  that  sends  repeated  or  incomplete  HTTP  requests  to  
exhaust
 
server
 
connection
 
pools.
 
GoldenEye
 
operates
 
by
 
initiating

connections
 
and

leaving

incomplete
 
or
 
sending
 
requests
 
at

designed
 
to
 
keep
 
server
 
threads
 
occupied,

legitimate
 
connections
 
from

served.
   
DoS  Slowloris:  A  slow  HTTP  attack  that  maintains  multiple  connections  to  a  web  server  by  
sending
 
partial
 
HTTP
 
headers
 
at
 
intentionally
 
slow
 
rates.
 
By

connections
 
open
 
indefinitely

periodic
 
header

connection

without

bandwidth

to

using

thresholds.
   
DoS  SlowHTTPTest:  Similar  to  Slowloris,  SlowHTTPTest  sends  HTTP  requests  very  slowly  
or
 
in

fragments
 
to
 
exhaust
 
server
 
resources.
 
The
 
attack
 
targets
 
the
 
server's

to
 
handle  slow  connections,  exploiting  timeout  configurations  that  allow  connections  to  remain  
open
 
for
 
extended
 
periods.

3.6  Handling  Class  Imbalance  
The  CICIDS2017  dataset  exhibits  severe  class  imbalance,  which  is  one  of  the  most  
significant
 
challenges
 
in
 
intrusion
 
detection.
 
Benign
 
traffic
 
constitutes

majority
 
of

minute
 
fractions

To

The

3:1

and

Strategy  2:  SMOTE  (Synthetic  Minority  Over-sampling  Technique)  
For

(Web

are

using

which
 
creates

instances

between

minority
 
class
 
instances
 
in
 
feature

     from  imblearn.under_sampling  import  RandomUnderSampler

         'Infiltration':  5000

         smote=SMOTE(sampling_strategy=sampling_strategy),     
         tomek=TomekLinks()     
     )     
     X_resampled,  y_resampled  =  smote_tomek.fit_resample(X_train_scaled,  
y_train_encoded)
    
          
 
  
Strategy  3:  Class  Weight  Adjustment  
During
 
model
 
training,

weights
 
are
 
inversely
 
proportional
 
to

frequencies,

misclassification
 
of

3.8  Model  Evaluation  Metrics  
Given  the  severe  class  imbalance  in  the  intrusion  detection  task,  accuracy  alone  is  an  
insufficient
 
performance
 
metric.
 
The
 
implementation
 
evaluates
 
models
 
using
 
a
 
comprehensive
 
suite
 
of
 
metrics:
   
Per-Class  Metrics:  ●  Precision:  True  Positives  /  (True  Positives  +  False  Positives)  —  measures  the  fraction  of  
predicted
 
attacks
 
that
 
are
 
actual
 
attacks
 ●  Recall  (Detection  Rate):  True  Positives  /  (True  Positives  +  False  Negatives)  —  measures  
the
 
fraction
 
of
 
actual
 
attacks
 
that
 
are
 
correctly
 
detected
 ●  F1-Score:  Harmonic  mean  of  Precision  and  Recall  —  balances  precision  and  recall  into  a  
single
 
metric
   
Aggregate  Metrics:  
●  Weighted  Average:  Metrics  weighted  by  class  support  (number  of  instances  per  class)  ●  Macro  Average:  Unweighted  average  across  all  classes  —  treats  all  classes  equally  
regardless
 
of
 
frequency
 ●  Overall  Accuracy:  Correct  predictions  /  Total  predictions    
Confusion  Matrix:  Provides  a  detailed  breakdown  of  correct  and  incorrect  classifications  for  
each
 
class
 
pair.
   
         
     from  sklearn.metrics  import  (accuracy_score,  precision_score,  
recall_score,
    
                                   f1_score,  classification_report,     
                                   confusion_matrix,  
ConfusionMatrixDisplay)
    
          
     def  evaluate_model(model,  X_test,  y_test,  label_encoder):     
         y_pred  =  model.predict(X_test)     
             
         accuracy  =  accuracy_score(y_test,  y_pred)     
         precision_weighted  =  precision_score(y_test,  y_pred,  
average='weighted')
    
         recall_weighted  =  recall_score(y_test,  y_pred,  
average='weighted')
    
         f1_weighted  =  f1_score(y_test,  y_pred,  average='weighted')     
             
         precision_macro  =  precision_score(y_test,  y_pred,  
average='macro')
    
         recall_macro  =  recall_score(y_test,  y_pred,  average='macro')     
         f1_macro  =  f1_score(y_test,  y_pred,  average='macro')     
             
         report  =  classification_report(y_test,  y_pred,     
                                        
target_names=label_encoder.classes_)
    
         cm  =  confusion_matrix(y_test,  y_pred)     
             
         return  {     
             'accuracy':  accuracy,     
             'precision_weighted':  precision_weighted,     
             'recall_weighted':  recall_weighted,     
             'f1_weighted':  f1_weighted,     
             'precision_macro':  precision_macro,     
             'recall_macro':  recall_macro,     
             'f1_macro':  f1_macro,     
             'classification_report':  report,     
             'confusion_matrix':  cm     
         }

6.2.3  Real-World  Generalization  Gap

communication)
 ●  Attack  tool  evolution  (newer  tools  may  generate  different  flow  patterns  than  those  used  in

 ●  Network  topology  complexity  (virtual  networks,  containers,  serverless  architectures)    
6.2.4  Computational  Constraints  for  Real-Time  Deployment  
While  the  Random  Forest  model  achieves  fast  inference  (~2ms  per  flow),  real-world  
deployment

●  Preprocessing  overhead  (feature  extraction,  scaling,  encoding)  adds  ~1-2ms  per  flow  ●  High-throughput  networks  generate  millions  of  flows  per  second,  requiring  parallel  
processing
 ●  Edge  deployment  scenarios  (IoT  devices,  network  switches)  have  severe  memory  and  
compute
 
constraints
 ●  Model  update  mechanisms  must  be  designed  to  avoid  service  interruption

claims.
 2.  No  zero-day  attack  evaluation:  The  supervised  learning  paradigm  cannot  detect  truly  
novel
 
attacks
 
not
 
represented
 
in
 
the
 
training
 
data.
 
Evaluating
 
the
 
model's
 
behavior
 
on
 
unknown
 
attack
 
types
 
would
 
provide
 
important
 
robustness
 
insights.
 3.  Static  feature  set:  The  implementation  uses  the  static  feature  set  provided  by  
CICFlowMeter.
 
Real-time
 
feature
 
extraction
 
from
 
live
 
network
 
traffic
 
introduces
 
additional
 
engineering
 
challenges
 
not
 
addressed
 
in
 
this
 
study.
 4.  No  adversarial  testing:  The  model's  robustness  against  adversarial  evasion  techniques  
(feature
 
manipulation,
 
traffic
 
morphing)
 
has
 
not
 
been
 
evaluated.
 5.  Limited  Infiltration  evaluation:  The  extremely  small  number  of  Infiltration  instances  in  
CICIDS2017
 
(36
 
total,
 
9
 
in
 
test
 
set)
 
makes
 
performance
 
claims
 
for
 
this
 
category
 
statistically
 
unreliable.
   
6.4  Future  Directions  
Based  on  the  findings  of  this  study,  the  following  future  research  directions  are  
recommended:
   
1.  Semi-supervised  and  unsupervised  approaches:  Combining  supervised  classification  
with
 
anomaly
 
detection
 
(e.g.,
 
autoencoders,
 
one-class
 
SVM)
 
to
 
detect
 
zero-day
 
attacks
 
without
 
labeled
 
training
 
data.
 2.  Federated  learning  for  IDS:  Training  models  across  distributed  network  environments  
without
 
sharing
 
raw
 
traffic
 
data,
 
addressing
 
privacy
 
concerns
 
while
 
improving
 
model
 
generalizability.
 3.  Federated  ensemble  architectures:  Developing  adaptive  ensemble  frameworks  that  
dynamically
 
weight
 
different
 
classifiers
 
based
 
on
 
real-time
 
performance
 
feedback,
 
rather
 
than
 
relying
 
on
 
static
 
model
 
selection.
 4.  Explainable  AI  integration:  Extending  the  SHAP-based  feature  importance  analysis  to  
provide
 
real-time
 
explanations
 
for
 
individual
 
detection
 
decisions,
 
enabling
 
security
 
analysts
 
to
 
understand
 
and
 
trust
 
model
 
outputs.
 5.  Cross-dataset  transfer  learning:  Investigating  transfer  learning  techniques  to  adapt  
models
 
trained
 
on
 
CICIDS2017
 
to
 
different
 
network
 
environments
 
and
 
newer
 
attack
 
datasets.
 6.  Edge-optimized  models:  Developing  lightweight  model  architectures  (e.g.,  pruned  
decision
 
trees,
 
quantized
 
neural
 
networks)
 
suitable
 
for
 
deployment
 
on
 
resource-constrained
 
IoT
 
devices
 
and
 
network
 
edge
 
devices.
 7.  Integration  with  threat  intelligence:  Incorporating  external  threat  intelligence  feeds  (IoC  
lists,
 
vulnerability
 
databases)
 
to
 
provide
 
contextual
 
enrichment
 
for
 
detection
 
decisions.
   
   
7.  Conclusion  
This  paper  presented  a  comprehensive  survey  and  practical  implementation  of  machine  
learning-based
 
intrusion
 
detection
 
systems
 
using
 
the
 
CICIDS2017
 
benchmark
 
dataset.
 
The
 
work
 
makes
 
two
 
primary
 
contributions:
   
First,  a  structured  survey  of  intrusion  detection  techniques  applied  to  five  major  attack  
categories—DoS,
 
DDoS,
 
Infiltration,
 
Brute-force,
 
and
 
Web-based
 
attacks—was
 
conducted,
 
examining
 
traditional
 
machine
 
learning
 
models,
 
ensemble
 
methods,
 
and
 
deep
 
learning
 
architectures
 
across
 
multiple
 
studies.
 
The
 
survey
 
revealed
 
that:
   
●  DDoS  attacks  consistently  achieve  the  highest  detection  accuracy  (95–99.97%)  due  to  
distinctive
 
volumetric
 
traffic
 
signatures
 ●  Web-based  attacks,  particularly  SQL  Injection  and  XSS,  present  the  greatest  detection  
challenge
 
(80–99%
 
variability)
 
due
 
to
 
their
 
subtle
 
flow-level
 
characteristics
 ●  Infiltration  detection  benefits  significantly  from  deep  learning  and  sequential  modeling  
techniques
 
(90–99%)
 ●  Ensemble  methods  (Random  Forest,  XGBoost)  generally  outperform  standalone  classifiers  
across
 
all
 
attack
 
categories
 ●  Binary  classification  settings  yield  consistently  higher  accuracy  than  multi-class  
configurations
 ●  Dataset  preprocessing,  feature  selection,  and  class  imbalance  handling  significantly  
influence
 
reported
 
performance
   
Second,  a  fully  implemented  and  deployed  AI-based  intrusion  detection  system  was  
developed,
 
trained
 
on
 
CICIDS2017,
 
and
 
made
 
publicly
 
accessible
 
as
 
a
 
web
 
application
 
at
 
intrusion.streamlit.app
.
 
The
 
implementation
 
incorporates:
   
●  Comprehensive  data  preprocessing  pipeline  handling  missing  values,  infinite  values,  and  
label
 
standardization
 ●  SHAP-based  feature  selection  reducing  the  feature  space  from  72  to  25  features  ●  Multi-strategy  class  imbalance  handling  using  SMOTE  and  random  undersampling  ●  Comparative  evaluation  of  five  machine  learning  models  (Random  Forest,  XGBoost,  
SVM,
 
Logistic
 
Regression,
 
k-NN)
 ●  Interactive  Streamlit  web  interface  with  real-time  prediction,  batch  classification,  EDA  
visualizations,
 
and
 
model
 
performance
 
dashboards
   
The  implemented  Random  Forest  model  achieved  an  overall  accuracy  of  98.62%,  with  
per-class
 
detection
 
rates
 
of
 
99.7%
 
for
 
DDoS,
 
99.2%
 
for
 
DoS,
 
98.8%
 
for
 
Brute
 
Force,
 
95.1%
 
for
 
Web
 
Attacks,
 
and
 
96.4%
 
for
 
Infiltration.
 
These
 
results
 
validate
 
the
 
survey
 
findings
 
and
 
demonstrate
 
that
 
ensemble
 
machine
 
learning
 
approaches
 
provide
 
robust,
 
deployable
 
intrusion
 
detection
 
capabilities
 
when
 
coupled
 
with
 
rigorous
 
data
 
preprocessing
 
and
 
feature
 
engineering.
   
The  implementation  validates  that  while  significant  progress  has  been  achieved  in  ML-based  
intrusion
 
detection,
 
challenges
 
remain—particularly
 
in
 
detecting
 
subtle
 
application-layer
 
attacks
 
and
 
extremely
 
rare
 
attack
 
categories.
 
No
 
single
 
model
 
universally
 
outperforms
 
others
 
across
 
all
 
attack
 
types,
 
reinforcing
 
the
 
importance
 
of
 
adaptive,
 
hybrid,
 
and
 
ensemble-based
 
intrusion
 
detection
 
frameworks.
 
The
 
publicly
 
available
 
code
 
and
 
live
 
deployment
 
provide
 
a
 
reproducible
 
baseline
 
for
 
future
 
research
 
and
 
a
 
practical
 
tool
 
for
 
cybersecurity
 
practitioners.

Appendix  A:  Glossary  of  Terms  
  
Term  Definition  IDS  Intrusion  Detection  System  —  a  system  that  monitors  network  traffic  for  malicious  activity  IPS  Intrusion  Prevention  System  —  an  IDS  with  automated  response  capabilities  DoS  Denial  of  Service  —  an  attack  that  makes  a  service  unavailable  by  exhausting  resources  DDoS  Distributed  Denial  of  Service  —  DoS  attack  originating  from  multiple  sources  (botnet)  XSS  Cross-Site  Scripting  —  injection  of  malicious  scripts  into  web  pages  SQL  Injection  Injection  of  malicious  SQL  queries  through  web  application  inputs  R2L  Remote  to  Local  —  attacks  where  a  remote  attacker  gains  local  access  CICFlowMeter  A  tool  for  extracting  flow-based  statistical  features  from  network  traffic  SHAP  SHapley  Additive  exPlanations  —  a  method  for  explaining  ML  model  predictions  SMOTE  Synthetic  Minority  Over-sampling  Technique  —  generates  synthetic  minority  class  samples  ADASYN  Adaptive  Synthetic  Sampling  —  an  adaptive  variant  of  SMOTE  SDN  Software-Defined  Networking  —  network  architecture  with  centralized  control  IoT  Internet  of  Things  —  network  of  interconnected  physical  devices  WAF  Web  Application  Firewall  —  filters  HTTP  traffic  to/from  web  applications  
Term  Definition  Botnet  A  network  of  compromised  computers  controlled  by  an  attacker  CNN  Convolutional  Neural  Network  —  a  deep  learning  architecture  for  spatial  pattern  extraction  LSTM  Long  Short-Term  Memory  —  a  recurrent  neural  network  for  sequential  data  BiLSTM  Bidirectional  LSTM  —  processes  sequences  in  both  forward  and  backward  directions
